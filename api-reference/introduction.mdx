## Bem-vindo

A **Social Go API** é um microsserviço interno projetado para ser consumido exclusivamente pelo backend principal da Conty. Ela fornece funcionalidades para sincronização e análise de métricas de redes sociais (Instagram e TikTok).

## Autenticação e Segurança

Todas as requisições devem ser autenticadas. Existem dois headers principais necessários para interagir com a API:

1. **Autenticação Interna**: Todas as chamadas devem incluir o header `X-Internal-Secret` para validar a origem da requisição.
2. **Contexto de Tenant**: Chamadas devem incluir `X-Enterprise-Id` para identificar o cliente e aplicar rate limiting.

```bash
X-Internal-Secret: SEU_SECRET_TOKEN
X-Enterprise-Id: 123e4567-e89b-12d3-a456-426614174000
```

## Rate Limiting

A API impõe limites de requisição para garantir estabilidade. Quando excedido, retorna status `429` com header `Retry-After`.

| Rota | Limite |
| :--- | :--- |
| Geral | 100 req/s |
| `/tiktok/download` | 5 req/min |
| `/video/enriched-metrics` | 10 req/min |

## Fluxo Assíncrono (Jobs)

Para operações pesadas (como download de mídia ou análise de IA), a API utiliza um padrão de Jobs assíncronos para evitar timeouts.

1. O cliente solicita a operação (ex: `POST /tiktok/download`).
2. A API retorna `202 Accepted` imediatamente com um `job_id`.
3. O cliente faz polling em `GET /jobs/{id}` até que o status seja `COMPLETED` ou `FAILED`.

## Fluxo de Uso

O ciclo de vida típico de uso da API envolve:

1. **Conexão**: Conecte uma conta de rede social usando `POST /connect`.
2. **Sincronização**: Acompanhe ou force a atualização de dados com `POST /accounts/{id}/sync`.
3. **Dashboard**: Consulte dados consolidados e métricas no `GET /accounts/{id}/dashboard`.
4. **Mídia**: Baixe e analise vídeos específicos via fluxo de Jobs.

## Padrões

- **Datas**: Formato ISO 8601 (`YYYY-MM-DDTHH:mm:ssZ`).
- **IDs**: UUID v4.
- **Erros**: Retornados em JSON no formato `{"code":"ERR_*","message":"..."}`.
