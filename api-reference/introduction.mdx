## Visão Geral dos Produtos

### AI Creators

API de chat conversacional que usa **GPT-4o** com fallback automático para buscar criadores de conteúdo.

| Endpoint | Método | Descrição |
|----------|--------|-----------|
| `/api/chat` | POST | Chat com streaming SSE |
| `/health` | GET | Health check |

**Base URL:** `https://ai-creators.appconty.com`

---

### Conty Match

Sistema de matching que encontra criadores ideais para campanhas usando busca semântica.

| Endpoint | Método | Descrição |
|----------|--------|-----------|
| `/match/find` | POST | Encontra criadores compatíveis |
| `/match/process-all` | POST | Processa todos os criadores |
| `/match/missions` | POST | Cria missões para criadores |
| `/match/notify` | POST | Envia notificações |
| `/creators` | GET | Lista criadores |
| `/creators/:id` | GET | Detalhes do criador |

**Base URL:** `https://match.appconty.com`

---

### Social API

Microsserviço para sincronização e análise de métricas de redes sociais.

| Endpoint | Método | Descrição |
|----------|--------|-----------|
| `/connect` | POST | Conecta conta social |
| `/accounts` | GET | Lista contas conectadas |
| `/accounts/:id/sync` | POST | Sincroniza dados |
| `/accounts/:id/dashboard` | GET | Dashboard de métricas |
| `/analytics` | POST | Análise de posts |
| `/tiktok/download` | POST | Download de vídeos |
| `/video/enriched-metrics` | POST | Métricas enriquecidas com IA |
| `/insights/*` | POST | Insights e predições |
| `/jobs/:id` | GET | Status de jobs assíncronos |

**Base URL:** `https://social.appconty.com`

## Autenticação

Cada produto tem seu próprio modelo de autenticação:

| Produto | Autenticação |
|---------|--------------|
| **AI Creators** | `session_id` no body (sem auth obrigatório) |
| **Conty Match** | API Key via header ou bearer token |
| **Social API** | `X-Internal-Secret` + `X-Enterprise-Id` |

### Social API (Headers obrigatórios)

```bash
X-Internal-Secret: SEU_SECRET_TOKEN
X-Enterprise-Id: 123e4567-e89b-12d3-a456-426614174000
```

## Padrões Comuns

Todas as APIs seguem os mesmos padrões:

| Padrão | Formato |
|--------|---------|
| **Datas** | ISO 8601 (`YYYY-MM-DDTHH:mm:ssZ`) |
| **IDs** | UUID v4 |
| **Erros** | JSON `{"error":"..."}` ou `{"code":"ERR_*","message":"..."}` |

---

## Rate Limiting

| Produto | Limite Geral | Limites Específicos |
|---------|--------------|---------------------|
| AI Creators | 60 req/min | - |
| Conty Match | 100 req/min | - |
| Social API | 100 req/s | `/tiktok/download`: 5/min, `/enriched-metrics`: 10/min |

Quando excedido, retorna status `429` com header `Retry-After`.

---

## Jobs Assíncronos (Social API)

Para operações pesadas, a Social API usa um padrão de jobs assíncronos:

1. Cliente solicita operação (ex: `POST /tiktok/download`)
2. API retorna `202 Accepted` com `job_id`
3. Cliente faz polling em `GET /jobs/{id}` até `COMPLETED` ou `FAILED`
